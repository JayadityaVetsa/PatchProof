# Documentation

This directory contains the detailed public specifications for PatchProof.

- `CLI_SPEC.md` — command behavior and exit codes.
- `CONFIGURATION.md` — `.patchproof.yml`.
- `ADAPTER_SPEC.md` — ecosystem integration contract.
- `REPORT_FORMAT.md` — canonical result and report semantics.
- `GITHUB_ACTION_SPEC.md` — Action inputs and security model.
- `REPOSITORY_STRUCTURE.md` — monorepo layout.
- `GLOSSARY.md` — canonical terminology.
- `COMPATIBILITY.md` — supported runtimes, frameworks, and platforms.
- `TROUBLESHOOTING.md` — common diagnostics and remediation.
- `DEMO.md` — reproducible build and proof demonstration.
- `decisions/` — accepted Architecture Decision Records.

Root-level documents describe product direction, architecture, execution, testing, and governance.
